# 🚀 4단계 - 영화 예매(프레임워크)

## 4단계 기능 목록

### 환경 및 인프라 설정
- [x] Spring Boot Web 관련 의존성 추가 (`spring-boot-starter-web`, Jackson 등)
- [x] 스프링 부트 애플리케이션 엔트리 포인트(`@SpringBootApplication`) 구성

### HTTP API 계층 (Controller & DTO) 구현
- [ ] 영화 및 상영 목록 조회 API (`GET /api/movies`)
  - [ ] 데이터베이스(또는 저장소)에 있는 전체 영화 목록을 조회하여 JSON 응답 구조로 반환
  - [ ] 각 영화별 상영 회차(Screening) 목록을 포함하며, 시작 시간(`startAt`)과 상영 길이를 바탕으로 계산된 종료 시간(`endAt`)을 함께 응답
- [ ] 영화 예매 생성 API (`POST /api/reservations`)
  - [ ] 상영 회차 ID(`screeningId`), 선택 좌석 리스트(`seats`), 사용 포인트(`usedPoints`), 결제 수단(`paymentMethod`)을 JSON 형식으로 요청받음
  - [ ] 요청 데이터를 기반으로 기존 예매 도메인 로직을 호출하여 예매 처리 및 최종 결제 금액 계산
  - [ ] 예매 성공 시 새로 생성된 예매 ID(`reservationId`), 입력받은 예약 내역, 결제 금액(`totalPrice`)을 포함해 `201 Created` 상태 코드로 응답
- [ ] 글로벌 예외 처리 (Exception Handling)
  - [ ] 클라이언트의 잘못된 요청에 대해 적절한 HTTP 상태 코드(예: `400 Bad Request`, `404 Not Found` 등)와 JSON 에러 응답을 일관되게 반환
  - [ ] (예: 형식이 안 맞는 요청, 기예약 좌석 선택, 없는 상영 회차 조회 등)

### 아키텍처 및 도메인 연동
- [ ] API 요청/응답을 위한 DTO(Data Transfer Object)를 정의하여 외부 인터페이스와 핵심 도메인 간의 결합도 분리
- [ ] 기존 코어 비즈니스 로직(1, 2단계에서 만든 영화 도메인, 할인/결제 정책 등)의 변경을 최소화하며 웹 계층과 자연스럽게 연결
- [ ] 트랜잭션 등 필요에 따라 서비스(Service) 계층을 도입하여 Controller와 Repository 사이의 흐름 제어

## 테스트 명세 (HTTP API)

### 환경 및 설정
- `@SpringBootTest`와 `RestTestClient`를 활용하여 실제 서버와 유사한 환경(Random Port)에서 End-to-End API 테스트 수행

### API 통합 테스트 (RestTestClient)
- 영화 목록 조회 API
  - 정상적으로 영화 목록을 조회하면 `200 OK` 상태 코드를 반환한다.
  - 응답 본문의 JSON 구조(`$.movies`)가 배열 형태이며, 기대하는 영화 정보와 계산된 종료 시간이 올바르게 포함되어 있는지 검증한다.
- 예매 생성 API
  - 올바른 요청 데이터로 예매를 진행하면 `201 Created` 상태 코드를 반환한다.
  - 응답 본문에 생성된 `reservationId`와 계산된 `totalPrice`가 누락 없이 올바르게 존재하는지 검증한다.
- 예외 상황 (실패 케이스) 테스트
  - 이미 예약이 완료된 좌석에 대해 예매를 요청하면 적절한 오류 상태 코드(예: `400 Bad Request`)를 반환한다.
  - 데이터베이스에 존재하지 않는 상영 회차(`screeningId`)로 예매를 요청하면 적절한 오류 상태 코드(예: `404 Not Found` 또는 `400`)를 반환한다.
  - JSON 요청 형식이 잘못되었거나 필수 값이 누락된 경우 적절한 상태 코드를 반환한다.

## 테스트 명세 (데이터베이스)

### 환경 및 설정

- 로컬 DB와 독립된 In-Memory H2 기반으로 테스트가 실행
- 각 테스트 실행 전후로 테이블 초기화 및 데이터 콜백이 이루어져 다른 테스트에 영향을 주지 않음

### Repository (DAO) 단위 테스트

- Connection 테스트: 설정된 JDBC URL을 통해 In-Memory 데이터베이스에 정상적으로 연결
- 영화 조회: 데이터베이스에 저장된 영화 정보를 정확히 조회하여 `Movie` 도메인 객체로 매핑
- 상영 회차 조회: 특정 영화와 날짜를 조건으로 쿼리 시, 올바른 상영 회차 정보들만 반환
- 예약 저장 및 조회:
    - 생성된 예약 객체를 저장소에 넘기면 예외 없이 INSERT
    - 저장된 예약 정보를 다시 조회했을 때, 저장했던 값과 동일한 상태로 복원
- 예약 좌석 처리:
    - 특정 상영 회차에 좌석을 저장한 후 조회 시, 해당 좌석들이 정상적으로 반환
    - 조인(JOIN) 쿼리를 통해 특정 예매 건에 속한 좌석 리스트를 정확히 가져옴

## 🚀 2단계 - 영화 예매(콘솔)

### 패키지 구조

- `model` — 핵심 비즈니스 로직 (1단계 도메인 객체)
- `view` — 사용자 인터페이스 (InputView, OutputView)
- `controller` — 흐름 제어 (MovieReservationController)

### 2단계 기능 목록

#### InputView 구현

- 예매 시작 여부 확인 입력 (Y/N)
- 영화 제목 입력
- 날짜 입력 (YYYY-MM-DD)
- 상영 번호 선택 입력
- 좌석 번호 입력 (예: A1, B2)
- 추가 영화 여부 확인 입력 (Y/N)
- 포인트 입력
- 결제 수단 선택 입력 (1: 신용카드, 2: 현금)
- 결제 확인 입력 (Y/N)

#### OutputView 구현

- 상영 목록 출력 (번호 포함)
- 좌석 배치도 출력 (등급 표시, 예약된 좌석 구분)

- 장바구니 전체 출력
- 상영 시간 겹침 오류 메시지 출력
- 최종 결제 금액 출력
- 예매 완료 내역 출력

#### MovieReservationController 구현

- 전체 예매 흐름 제어 (영화 선택 → 상영 선택 → 좌석 선택 → 결제)
- 장바구니 방식: 여러 영화를 담은 후 일괄 결제
- 상영 시간 겹침 시 재선택 요청
- 존재하지 않는 영화 입력 시 재입력 요청
- 할인 정책 적용 (무비데이 → 시간 순서)
- 포인트 차감 후 결제 수단 할인 적용

## 1단계 기능 및 작업 명세

### 금액 및 포인트 구현

금액을 포장하는 값 객체를 구현한다. 0 미만의 금액은 허용하지 않는다. 포인트도 같은 방식으로 포장하며, 음수가 될 수 없고 금액으로 변환할 수 있다.

### 영화 정보 값 객체 구현

상영 길이를 분 단위로 포장한다. 0 이하는 허용하지 않는다. 상영 가능 기간은 시작일과 종료일로 표현하며, 시작일이 종료일보다 이후일 수 없다. 특정 날짜가 기간 안에 포함되는지
확인할 수 있다. 좌석 번호는 행(A~E)과 열(1~4)의 조합으로 표현하며 유효 범위를 검증한다. 좌석 등급은 S, A, B 세 가지이며 각 등급의 가격은 각각 18,000원,
15,000원, 12,000원이다.

### 영화 및 좌석 구현

영화는 제목, 상영 길이, 상영 가능 기간을 가진다. 좌석은 좌석 번호와 등급만을 가지는 순수 값 객체다. 예약 여부와 같은 가변 상태를 보유하지 않는다.

### 좌석 목록 구현

좌석 목록을 일급 컬렉션으로 관리한다. 좌석 번호로 특정 좌석을 찾을 수 있고, 이미 예약된 좌석 번호 집합을 제외한 예약 가능 좌석 목록을 반환할 수 있다. 선택된 좌석들의 등급
가격 합계를 계산하며, 좌석 번호 목록을 반환할 수 있다.

### 상영관 구현

상영관은 상영관 이름과 물리적 좌석 레이아웃을 가진다.

### 상영 회차 구현

상영 회차는 특정 영화, 시작 시각, 상영관을 가진다. 종료 시각은 생성자 파라미터로 받지 않으며, 영화의 상영 길이로부터 계산된다. 해당 회차에서 예약된 좌석 번호를 관리하며,
좌석 예약과 예약 가능 좌석 조회를 담당한다.

좌석을 예약할 때는 중복 선택, 이미 예약된 좌석, 존재하지 않는 좌석을 모두 검증한다. 검증을 통과하면 예약 객체를 반환한다. 예약 객체를 생성하는 유일한 진입점이다.

### 상영 회차 목록 구현

상영 회차 목록을 일급 컬렉션으로 관리하며 순회할 수 있다. 목록이 비어 있는지 확인할 수 있고, 영화와 날짜 기준으로 필터링할 수 있다.

### 예약 구현

예약은 상영 회차와 선택된 좌석 목록을 보관한다. 영화 제목, 시작 시각, 예매 기본 가격을 제공한다.

### 예약 목록 구현

예약 목록을 일급 컬렉션으로 관리하며 순회할 수 있다. 빈 상태로 시작할 수 있다. 예약을 추가할 때 기존 예약과 상영 시간이 겹치는지 검증하며, 겹치면 예외가 발생한다. 추가 시
새로운 목록을 반환하는 불변 방식으로 동작한다.

### 영화 목록 및 스케줄러 구현

영화 목록을 일급 컬렉션으로 관리한다. 전체 목록에 특정 영화 포함 여부를 확인하고, 제목으로 영화를 검색할 수 있다.

스케줄러는 모든 상영 회차 정보를 내부에 보유한다. 영화와 날짜를 받아 해당 상영 회차 목록을 반환하고, 전체 영화 목록도 제공한다.

### 할인 정책 구현

할인 정책 sealed 인터페이스를 정의한다. 무비데이 할인 정책은 매월 10일, 20일, 30일에 10% 비율로 할인한다. 시간 할인 정책은 오전 11시 이전 또는 오후 8시
이후(정각 포함) 상영 시 2,000원을 할인한다.

### 결제 수단 구현

결제 수단 sealed 인터페이스를 정의한다. 신용카드는 5%, 현금은 2% 추가 할인을 적용한다.

### 결제 시스템 구현

할인 정책 적용 → 포인트 차감 → 결제 수단 할인 순서로 최종 결제 금액을 계산한다. 할인 정책은 리스트 순서대로 적용하며, 무비데이 → 시간 순서를 호출부에서 보장한다. 결제
결과는 최종 결제 금액과 사용된 포인트를 담은 객체로 반환한다.

## 테스트 명세

### 단위 테스트

#### 금액 및 포인트

- 초기 금액이 올바르게 생성된다
- 금액이 0 미만이면 예외가 발생한다
- 금액이 0 미만일 때 올바른 예외 메시지가 출력된다
- 포인트가 음수이면 예외가 발생한다
- 포인트를 금액으로 올바르게 변환한다

#### 영화 정보 값 객체

- 상영 길이가 0 이하이면 예외가 발생한다
- 시작일이 종료일보다 이후이면 예외가 발생한다
- 특정 날짜가 상영 가능 기간에 포함되면 true를 반환한다
- 특정 날짜가 상영 가능 기간에 포함되지 않으면 false를 반환한다
- 유효한 행과 열로 좌석 번호가 생성된다
- 행이 유효 범위를 벗어나면 예외가 발생한다
- 열이 유효 범위를 벗어나면 예외가 발생한다
- S등급의 가격은 18,000원이다
- A등급의 가격은 15,000원이다
- B등급의 가격은 12,000원이다

#### 좌석 및 좌석 목록

- 동일한 좌석 번호와 등급을 가진 두 좌석은 동등하다
- 등급이 다른 두 좌석은 동등하지 않다
- 선택한 좌석들의 등급 가격이 올바르게 합산된다
- 예약된 좌석 번호를 제외한 나머지 좌석을 반환한다
- 존재하지 않는 좌석을 찾으면 예외가 발생한다

#### 영화 목록

- 영화 목록이 비어있으면 예외가 발생한다
- 영화 목록에 찾는 영화가 있으면 true를 반환한다
- 영화 목록에 찾는 영화가 없으면 false를 반환한다
- 제목으로 영화를 찾을 수 있다
- 존재하지 않는 제목으로 찾으면 null을 반환한다

#### 상영 회차

- 종료 시각은 시작 시각에 영화 상영 길이를 더한 값이다
- 시작 시각과 상영 날짜가 올바르게 반환된다
- 두 상영 회차의 시간이 겹치면 true를 반환한다
- 두 상영 회차의 시간이 겹치지 않으면 false를 반환한다
- 이미 예약된 좌석을 다시 예약하면 예외가 발생한다
- 한 예약 요청 내에서 중복된 좌석을 선택하면 예외가 발생한다
- 존재하지 않는 좌석을 예약하면 예외가 발생한다
- 정상 예약 후 예약 가능 좌석 조회 시 해당 좌석이 제외된다
- 예약 결과의 기본 가격이 올바르게 계산된다

#### 예약 목록

- 빈 상태로 시작할 수 있다
- 시간이 겹치는 상영을 예매하면 예외가 발생한다
- 시간이 겹치지 않는 상영을 예매할 수 있다

#### 스케줄러

- 해당 영화의 상영 회차가 없으면 빈 상영 회차 목록을 반환한다
- 영화와 날짜에 맞는 상영 회차 목록을 반환한다
- 전체 영화 목록을 반환한다

#### 할인 정책

- 10일, 20일, 30일에 각각 10% 할인이 적용된다
- 해당하지 않는 날짜에는 할인이 적용되지 않는다
- 오전 11시 이전 상영에 2,000원 할인이 적용된다
- 오후 8시 이후 상영에 2,000원 할인이 적용된다
- 오후 8시 정각 상영에 2,000원 할인이 적용된다
- 해당하지 않는 시간대에는 할인이 적용되지 않는다

#### 결제 수단

- 신용카드 결제 시 5% 할인이 적용된다
- 현금 결제 시 2% 할인이 적용된다

#### 결제 시스템

- 무비데이 할인이 시간 할인보다 먼저 적용된다
- 포인트가 예매 금액에서 올바르게 차감된다
- 포인트 차감 후 결제 수단 할인이 적용된다
- 신용카드 결제 시 5% 추가 할인이 적용된다
- 현금 결제 시 2% 추가 할인이 적용된다
